# WithCamp Map

## 실행

관리자 모드에서 변경한 카메라, 도면, 미디어 설정을 다른 사용자에게도 적용하려면 Node 서버로 실행해야 합니다.

```bash
npm start
```

```text
http://localhost:4173
```

관리자 변경 사항은 `data/map-state.json`에 저장됩니다. 정적 서버로만 열면 앱은 계속 동작하지만 저장 내용은 해당 브라우저의 `localStorage`에만 남습니다.

## Vercel 배포

GitHub에 커밋된 `data/map-state.json`은 Vercel의 `/api/state` 함수가 읽어 공개 화면에 반영합니다.

Vercel 배포 환경은 실행 중인 앱이 저장소 파일을 영구 수정할 수 없으므로, admin에서 수정한 내용을 모든 사용자에게 즉시 공유 저장하려면 별도 영구 저장소가 필요합니다. 저장소가 없으면 변경 내용은 현재 브라우저에는 남지만 다른 사용자에게 공유되지 않습니다.

건물 1층/2층 도면을 나란히 보여주고, 도면 위 카메라 버튼을 통해 방별 이미지/영상 미디어를 확인하는 정적 웹앱입니다.

## 실행

빌드 도구가 필요 없습니다. 아래 명령으로 로컬 서버를 열고 브라우저에서 접속하세요.

```bash
python3 -m http.server 4173
```

```text
http://localhost:4173
```

## 주요 기능

- 1층과 2층 도면을 나란히 표시
- 도면 위 카메라 버튼 클릭 시 해당 위치의 이미지/영상 썸네일 표시
- 썸네일 클릭 시 별도 뷰어 창에서 이미지 표시 또는 영상 재생
- 관리자 모드에서 카메라 버튼 추가/삭제
- 관리자 모드에서 선택 카메라의 이미지/영상 미디어 추가/삭제
- 관리자 모드에서 도면 이미지 업로드 또는 URL 교체
- 변경 데이터는 브라우저 `localStorage`에 저장

## 영상 URL 주의사항

앱 안에서 바로 재생하려면 아래 형식 중 하나를 등록하세요.

- YouTube 퍼가기 소스: `<iframe src="https://www.youtube.com/embed/..."></iframe>`
- YouTube 일반 URL: `https://www.youtube.com/watch?v=...`, `https://youtu.be/...`
- 직접 영상 파일 URL: `.mp4`, `.webm`

Google Photos 공유 링크처럼 웹페이지로 열리는 주소는 앱 안의 플레이어에서 직접 재생되지 않으며, 뷰어에서 새 탭 열기 링크로 안내됩니다.

## 운영 적용 시 권장 구조

현재 구현은 프론트엔드 프로토타입입니다. 실제 운영에서는 아래 항목을 서버로 분리하는 것이 좋습니다.

- 관리자 로그인/권한
- 도면, 이미지, 영상 파일 업로드 및 스토리지 저장
- 카메라 좌표와 미디어 목록 DB 저장
- 미디어 URL 서명, 접근 제어, 삭제 이력 관리
